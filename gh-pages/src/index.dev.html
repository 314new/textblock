Textblock.js

Textblock sets responsive type beyond standard CSS breakpoints. It adjusts size and leading to cast continuously scaled type across varying screen widths. It works on top of your current CSS as a progressive enhancement.

Download from Github

or npm install textblock

1
Add textblock.js to the head or foot of your document
2
Target elements in a Textblock( ) script using js notation
3
Adjust font sizes and line heights

Textblock.js uses minimum and maximum values for font size, leading (line height), and measure (container width). The script calculates floating point values based on units defined within the document head (or foot):

  • Minimum/maximum font size
  • Minimum/maximum leading
  • Minimum/maximum measure

For easy math set the root em to 10px so em units are base 10, i.e. 2.4em = 24px. But any number will do because once you start adjusting the min/max numbers the experience is more visual than punching numbers into a calculator. And for us gridnicks, textblock gives you absolute control over calculating modular scales, in floating points.

To Initialize

Textblock([{
 target:".some-element"
}]);

Optional Parameters

  • target: The element that should be resized ".your‑class", "#cool-id p" (required  )
  • minWidth: default 280
  • maxWidth: default 800
  • minFontSize: default 1.9
  • maxFontSize: default 2.6
  • minLineHeight: default 1.33
  • maxLineHeight: default 1.25
  • container: The container width to measure. Defaults to "parent" and can alternately be set to "self".
  • units: default "em"

Example Including Extra Parameters

Textblock([{
 target:".some-class",
 minWidth: 280,
 maxWidth: 800,
 minFontSize: 1.9,
 maxFontSize: 2.6,
 minLineHeight: 1.33,
 maxLineHeight: 1.25,
 container: "self",
 units: "rem"
}]);

Multiple Elements with Different Settings

Textblock([
 {
  target:".some-class",
  minFontSize: 1.9,
  maxFontSize: 2.6,
 },
 {
  target:".another",
  minFontSize: 2.4,
  maxFontSize: 3.6,
  container: "self"
 }
]);
